Google Transit Feed Import Add-In

 

This add-in allows you to import public transport network and timetable data from a Google Transit feed, an open format used by transit operators to submit their service data to Google’s transit trip planner. The specification of the format can be found here: http://code.google.com/transit/spec/transit_feed_specification.html.

 

Method

 

The importer reads the individual files that make up a Google Transit Feed data set and creates corresponding objects in VISUM that describe the public transport supply. The importer uses most, but not all of the information in the feed data files. VISUM creates a calendar, valid days, the stop hierarchy and the line hierarchy (down to vehicle journeys). Currently, the following data are not transferred: fare definitions, shapes between stops, blocks.

 

The creation of ValidDays requires the presence of the Calendar additional module. This dependency will lead to an error message if the current script is run without the Calendar module. If your license does not include the Calendar additional module and you click OK to the error message, then the result of the import will still be usable. The only limitation will be that all vehicle journeys will be marked as running daily whereas the original data may have specified running day restrictions. See the date filter in the parameter description below for a workaround.

 

Also the importer currently assumes that the input files are plain ASCII, although the Google Transit Feed specification mentions that it could be some flavor of Unicode.

 

Parameters and Usage

 

Start by placing all of the files in the Google Transit Feed data set in a single directory. Unzip them, if they are contained in a ZIP archive. They should be individual text files. Then start the add-in.

 

 

Source directory: Select the directory containing the files.

 

Use filter date: If you select this option, VISUM will only import vehicle journeys which run on the specified date. This option gives you a workaround, if your license does not include the Calendar additional module, but you still want separate VISUM models for, say, a typical Monday through Friday, Saturday, and Sunday, respectively. For this example, run the importer three times and apply the date filter with a specific dates of one Monday, one Saturday, and one Sunday. The import will show all vehicle journeys as daily (because you do not have the Calendar module), but the models will only contain the vehicle journeys that run on each of the three representative dates. Finer distinctions of running days will be lost, of course.

 

 

Caveat

 

The Google Transit Feed specification is subject to change and the current add-in conforms to the specification as of early October 2007. While we hope that future extensions of the specification will not break the importer, we cannot exclude the possibility. Please, report incompatibilities to hotline.visum@ptv.de.
